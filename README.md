# CortexOS Management Server

Centralized fleet management for multiple CortexOS Server instances. Monitor health, manage access, and control your entire CortexOS infrastructure from a single dashboard.

## Architecture

```
┌─────────────────────────────────┐
│   Management Dashboard (HTML)    │
│   http://localhost:9443          │
├─────────────────────────────────┤
│   Express.js API (/api/v1/)     │
│   JWT Auth + TOTP 2FA           │
├─────────────────────────────────┤
│   SQLite (better-sqlite3)       │
│   Users, Servers, Health, Audit │
└─────────────┬───────────────────┘
              │ Health Poller (30s)
    ┌─────────┴─────────┐
    ▼                   ▼
┌────────┐        ┌────────┐
│Server 1│        │Server 2│  ...
│/stats  │        │/stats  │
└────────┘        └────────┘
```

## Quick Start

```bash
# Clone & install
cd /path/to/cortex-management-server
npm install

# Start (dev mode with auto-reload)
npm run dev

# Start (production)
npm start
```

Dashboard: **http://localhost:9443/dashboard/**

Default credentials: **admin / admin**

> ⚠️ Change the default password after first login!

## Install as Service

```bash
sudo bash scripts/install-management.sh
```

This will:
- Install Node.js if needed
- Run `npm install`
- Create & start a `cortex-management` systemd service
- Enable auto-start on boot

## API Endpoints

### Authentication

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/v1/auth/login` | Login → JWT token |
| POST | `/api/v1/auth/logout` | Logout |
| GET | `/api/v1/auth/me` | Current user info |
| PUT | `/api/v1/auth/password` | Change password |
| POST | `/api/v1/auth/2fa/setup` | Generate TOTP secret |
| POST | `/api/v1/auth/2fa/verify` | Verify & enable 2FA |
| POST | `/api/v1/auth/2fa/disable` | Disable 2FA |
| GET | `/api/v1/auth/users` | List users (admin) |
| POST | `/api/v1/auth/users` | Create user (admin) |
| DELETE | `/api/v1/auth/users/:id` | Deactivate user (admin) |

### Fleet Management

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/servers` | List all servers + health |
| POST | `/api/v1/servers` | Add server manually |
| GET | `/api/v1/servers/:id` | Server detail |
| DELETE | `/api/v1/servers/:id` | Remove server |
| GET | `/api/v1/servers/:id/health` | Health history |
| POST | `/api/v1/servers/register` | Register via install token |

### Tokens

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/v1/tokens` | Generate install token |
| GET | `/api/v1/tokens` | List active tokens |
| DELETE | `/api/v1/tokens/:id` | Revoke token |

### Audit

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/audit` | Audit log (admin) |

## Configuration

Config is auto-generated on first run at `data/config.json`:

```json
{
  "jwtSecret": "<auto-generated>",
  "port": 9443
}
```

Database is stored at `data/cortex-management.db` (SQLite).

## Dashboard Features

- **Fleet View** — Server cards with live CPU/RAM/disk bars
- **Server Detail** — Full stats, health history charts, link to server dashboard
- **Add Server** — Generate install tokens or add manually
- **User Management** — Create/deactivate users, role assignment
- **2FA** — Optional TOTP two-factor authentication
- **Audit Log** — Track all actions across the system
- **Auto-refresh** — Fleet view updates every 30 seconds

## Tech Stack

- **Runtime:** Node.js (ESM)
- **Server:** Express.js
- **Database:** SQLite via better-sqlite3
- **Auth:** bcryptjs + jsonwebtoken + otplib (TOTP)
- **Dashboard:** Single HTML file (no build step)
